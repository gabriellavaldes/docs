# Overview

This guide will walk you through the process of launching the [Rails Getting Started example](https://guides.rubyonrails.org/v4.2.7/getting_started.html) on Aptible. Deploy a starter template Node.js app using the Express framework on Aptible

# Get started

<Tabs>
  <Tab title="Deploy via Template">
  Using the [Deploy](https://app.aptible.com/create) tool in the Aptible Dashboard, you can deploy the **Ruby on Rails Template.**
  
  <AccordionGroup>
  <Accordion title="Deploy with Git Push">
  Select Deploy with Git Push
  </Accordion>
  <Accordion title="Add an SSH key">
  If you have not done so already, you will be prompted to set up an [SSH key](https://www.aptible.com/docs/public-key-authentication#ssh-keys).
  </Accordion>
  <Accordion title="Configure Environment">
  Select your [stack](https://www.aptible.com/docs/stacks) to deploy your resources. This will determine what region your resources are deployed to. Then, name the [environment](https://www.aptible.com/docs/environments) your resources will be grouped into.
  </Accordion>
<Accordion title="Prepare the template">
Select `Ruby on Rails Template` for deployment, and follow command-line instructions.
</Accordion>
<Accordion title="Fill environment variables">
Aptible will automatically fill this template's required databases, services, and app's configuration with environment variable keys for you to fill with values. Once complete, save and deploy the code!
</Accordion>
<Accordion title="View logs in real time">
</Accordion>
<Accordion title="Expose your app to the internet">
Now that your code is deployed, it's time to expose your app to the internet. Select the service that needs an [endpoint](https://www.aptible.com/docs/endpoints), and Aptible will automatically provision a managed endpoint.
</Accordion>
